## 오늘 공부한 내용

-Feature Branch Workflow
Shared repository model
사용자가 원격저장소의 소유권을 공유 받는 방식 

>"중앙 원본 저장소는 하나, 기능 개발은 별도의 브랜치에서"

`git push origin master`
원격 저장소에 push 할거다, origin이라는 이름의 원격저장소에, master브랜치를.

`git push origin feature/login`
원격 저장소에 push 할거다, origin이라는 이름의 원격저장소에, feature/login브랜치를.
작업이 완료되면, 중앙 저장소에 브랜치를 푸시(push)하고 풀 리퀘스트(Pull Request, PR)를 통해 
master 등 브랜치에 합병(Merge)해 달라 공식적으로 요청하는 기능
이는 단순 코드 합치기 요청을 넘어, 협업을 위한 핵심 소통 도구로 사용
>PR의 핵심 목적: 코드 리뷰(Code Review)
PR의 진정한 목적은 코드를 합치기 전에, 동료나 프로젝트 관리자가 변경된 내용을 검토(Review)하고 피드백을 줄 수 있는 공식적인 장을 마련하는 것

원격 저장소 공유

클론 받고 각자 브랜치 생성 및 기능 구현
브랜치 올라가면 Pull Request 올라가게 됨
원격 저장소 병합 등으로 master 업데이트되면 master로 switch- 병합된 master의 내용을 pull
원격 저장소에서 병합 완료 된 로컬 브랜치 삭제
`git switch master`
`git pull origin master`
`git branch -D feature/login`


-Forking Flow 
Fork 다른 사용자의 원격 저장소를 자신의 계정으로 복사하는 것
>"내 저장소에서 자유롭게 작업하고, 괜찮으면 원본에 제안(PR)하가"
중앙 원본 저장소(Upsteram)에 직접 푸시(push) 할 수 없음
소유권이 없는 원격 저장소를 fork를 통해 복제
추후 로컬 저장소를 원본 원격 저장소와 동기화하기 위해 URL을 연결
기능 추가를 위해 branch 생성 및 기능 구현 
복제(origin) 풀 리퀘스트를 통으로 upstream에 보냄

## 테스트 기법
AI는 코드를 완벽하게 작성할까?
여러 번의 기능 추가 및 반복적인 수정을 하다보면, 기존의 잘 만들어진 기능이 망가지는 경우가 생김
회귀 버그/ like 옆방 배선 건드려서 불이 안 켜지는데 옆방을 안 가서 확인을 못함
AI가 작성한 코드를 어떻게 검증할 수 있을까? 
1. 수많은 기능 추가 및 수정이 일어날 때마다 기존 기증이 잘 동작하는지 직접 테스트
   -시간부족, 테스트 항목 놓칠 수 있음
2. 최종 개발이 다 끝나고 테스트하기
  - 코드가 엉켜 모든 기능이 꼬여버릴 수 있음
  - 언제부터 코드가 망가졌는지 파악하기 어려움
  
  검증을 컴퓨터에게 맡기자

컴퓨터에게 기능 단위로 검증시키는 방법
모든 기능들이 정상적으로 동작하는지 편하게 테스트할 수 있음
테스트 코드가 동일한 입력을 넣었을 때, 동일한 출력값을 확인하는 것

1.준비단계 2.실행 3.검증
AAA 패턴 3단계를 반드시 실행한다

Unit Test 실습: 계산기 
1. 계산기를 만든다.
2. 테스트 코드를 붙인다.
3. 새 기능을 추가한다.
4. 일부로 기능을 망가지게 해서 테스트 코드를 확인.

Unit Test 주의사항 
테스트 코드를 통과했다는 것이 버그가 없다는 것이 아님
우리가 짠 테스트코드는 작성한 기능에 대해서만 테스트를 해주는 것 뿐
내가 놓친 테스트에 대해서는 여전히 에러가 발생할 수 있음
테스트 코드를 너무 맹신하지 말 것
- 테스트 코드 작성에 많은 시간을 쏟지 말 것
  오류가 발생하면 안되는 핵심 로직에만 테스트 코드를 작성할 것

  테스트를 꼭 붙여야 하는 곳
  ex 포인트 제도(계산)
  게시글 삭제 조건(데이터 삭제 조건)
  로그인/권한 판별
  
  테스트를 안 붙여도 되는 곳
  ex 버튼 색깔, 폰트 크기
  화면이 움직이는 애니메이션

  테스트 커버리지 - 작성된 테스트 케이스에서 우리의 전체 코드 중 어느 정도 비율로?
  1. 게시글 등록
  2. 게시글 삭제
  3. 챗봇에 메시지를 보내면 응답 형식이 맞게 오는가
